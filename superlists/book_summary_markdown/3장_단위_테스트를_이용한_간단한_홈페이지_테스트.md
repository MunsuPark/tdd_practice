# 3장 단위 테스트를 이용한 간단한 홈페이지 테스트

## 기능테스트 vs 단위테스트

기능 테스트: 사용자 관점에서 애플리 케이션 외부를 테스트

단위 테스트: 프로그래머 관점에서 그 내부를 테스트

## TDD 접근법

1. 기능 테스트를 작성해서 사용자 관점의 새로운 기능성을 정의
2. 기능 테스트가 실패하고 나면 어떻게 코드를 작성해야 테스트를 통과시킬지 고민하고, 하나 또는 그 이상의 단위 테스트를 이용해서 어떻게 코드가 동작해야 하는지 정의
3. 단위 테스트가 실패하고 나면 단위 테스트를 통과할 수 있을 정도의 최소한의 코드만 작성
4. 기능 테스트 재실행하여 제대로 동작하는지 확인하고 필요하다면 새로운 단위 테스트 작성

## Django의 주요 요청 처리 흐름

1. 특정 URL에 대한 HTTP 요청을 받는다.
2. Django는 특정 규칙을 이용해서 해당 요청에 어떤 뷰 함수를 실행할지 결정한다.(URL 해석)
3. 해당 뷰가 요청을 처리해서 HTTP 응답으로 반환한다.

## urls.py

Django는 urls.py를 통해 어떻게 URL을 뷰 함수에 맵핑할지 정의.

## 뷰를 위한 단위 테스트

HTML 형식을 실제 응답을 반환하는지 확인하는 테스트 작성

### 단위 테스트 - 코드 주기

1. 터미널에서 단위 테스트를 실행해서 어떻게 실패하는지 확인
2. 편집기상에서 현재 테스트를 수정하기 위한 최소한의 코드를 변경
